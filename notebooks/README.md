# Notebooks — Exploration Only

These are for poking at data and prototyping. **They are not run in CI** and should never be imported from `models/`, `api/`, or anywhere else.

When a notebook produces something worth keeping (a transform, a model architecture, a feature), promote it to a proper `.py` file under the relevant owner-folder and add tests.

Suggested naming: `YYYY-MM-DD_initials_topic.ipynb` — e.g. `2026-06-03_van_distilbert_lr_sweep.ipynb`.
